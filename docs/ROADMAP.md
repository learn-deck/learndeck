# Roadmap

## Now

Validate the first session with real learners and strengthen the DDD course as teaching.

## Next

Make setup resilient and self-explanatory, then add a second course, Testing Fundamentals, to prove the format generalises.

## Later

Add catalogue provenance and versioning, followed by packaging and distribution improvements.

Roadmap changes follow observed learner outcomes rather than feature accumulation.
Not building now: in-app AI chat—because it would outsource the learner’s thinking; multi-language course paths—because we are validating one focused path first; a course marketplace—because the catalogue stays curated; cloud accounts/analytics—because local-first privacy comes first; app-store packaging—because the self-hosted Bun app comes first; and AI-written answers—because the guide should support thinking, not replace it.
