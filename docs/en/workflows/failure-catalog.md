# Failure catalog

| Failure                                                  | Required behavior                                                                                                                                                                                                                                                         | Forbidden shortcut                                                                                                                                             |
| -------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Lease expires                                            | At authoritative `now >= expiresAt`, end the lease as `LEASE_EXPIRED`, reject every owner action, and authorize another attempt only if budget remains.                                                                                                                   | Treating equality as still active or accepting a late action because the runner once held the lease.                                                           |
| Duplicate artifact submission                            | Return the recorded result for the same attempt and digest.                                                                                                                                                                                                               | Creating a second artifact effect.                                                                                                                             |
| Conflicting resubmission                                 | Reject a different digest after a terminal submission.                                                                                                                                                                                                                    | Replacing audit history.                                                                                                                                       |
| Stale mission, artifact, or gate identity                | Reject the mismatched input.                                                                                                                                                                                                                                              | Merging or silently upgrading versions.                                                                                                                        |
| Forbidden-path patch                                     | Workshop accepts and publishes a topologically valid artifact; the later trusted Verification `check-allowed-scope` gate fails it deterministically.                                                                                                                      | Rejecting it during Workshop submission or letting the producing runner waive the verification rule.                                                           |
| Mandatory gate fails or times out                        | Produce a failed verdict with bounded evidence.                                                                                                                                                                                                                           | Agent override of a mandatory gate.                                                                                                                            |
| Producing runner attempts verification                   | Reject the verifier assignment.                                                                                                                                                                                                                                           | Self-verification.                                                                                                                                             |
| Verifier crashes                                         | Resume or replay idempotently for the same inputs.                                                                                                                                                                                                                        | Starting a logically different run without audit linkage.                                                                                                      |
| Verification infrastructure cannot produce a gate result | Publish `verification.aborted.v1`; when `retryable` and budget remains, authorize a fresh bounded attempt, otherwise require human decision. Cancellation is non-retryable and follows cancellation.                                                                      | Resuming the terminal run, reporting a failed gate, or issuing a completion recommendation.                                                                    |
| Attempt budget exhausted                                 | Enter `NEEDS_HUMAN_DECISION`.                                                                                                                                                                                                                                             | Infinite automatic retry.                                                                                                                                      |
| Human rejects passing work                               | Preserve evidence; optionally authorize a bounded new attempt with feedback.                                                                                                                                                                                              | Treating technical pass as approval.                                                                                                                           |
| Mission cancelled                                        | Revoke a current `READY` or `LEASED` attempt and reject new attempts while preserving history.                                                                                                                                                                            | Mapping authority revocation to owner-only abandon/fail, or deleting attempts or evidence.                                                                     |
| Message redelivered after commit                         | Detect prior processing and acknowledge without another business effect; for lease response loss, return the exact confidential typed response and original token for the same request fingerprint.                                                                       | Assuming exactly-once delivery, creating another lease, or persisting/publishing/logging the raw token in aggregate or event state.                            |
| Invalid or poison event                                  | Keep it observable in a DLQ. Replay only through bearer-authenticated `replayDeadLetter` with idempotency, operator identity and reason, optimistic revision and original-digest checks, a distinct nested-envelope ID, contract validation, and immutable audit linkage. | Dropping it, looping forever, accepting redundant replacement identity/type fields, mutating the original envelope, or replaying directly from broker tooling. |
| Telemetry unavailable                                    | Continue according to durable domain state.                                                                                                                                                                                                                               | Using traces as workflow state.                                                                                                                                |

Each case is an independent acceptance probe with explicit initial state; the
table is not one sequential attempt history. These cases are acceptance
requirements. Infrastructure-specific mechanisms are chosen by each stack
without changing the business outcomes.
